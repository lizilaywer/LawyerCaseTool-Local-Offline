# 法律文书离线识别系统设计方案

## 一、项目概述

基于规则的离线本地法律文书识别系统，支持各类法律文书的结构化信息提取。

**核心特点：**
- ✅ 纯离线运行，无需联网
- ✅ 无需AI模型，基于规则引擎
- ✅ 响应快速，毫秒级解析
- ✅ 隐私安全，数据不上传
- ✅ 支持10+种法律文书类型

---

## 三、各类型文书详细规律

### 3.1 民事判决书

#### 标准结构
```
××××人民法院
民事判决书
(××××)×民初××号

[当事人信息]
原告：×××，性别，出生日期，民族，住址...
委托诉讼代理人：×××，律师事务所...
被告：×××...

[案件由来]
原告×××与被告×××...一案，本院于××××年××月××日立案后...

[诉辩意见]
原告×××向本院提出诉讼请求：...
事实与理由：...
被告×××辩称：...

[法院审理]
经审理查明：...
本院认为：...

[判决结果]
判决如下：
一、...
二、...

[尾部]
案件受理费...
如不服本判决...
审判长×××
××××年××月××日
```

#### 关键识别特征
- **标题**: "××人民法院" + "民事判决书"
- **案号**: (年份)法院代字+"民初"+编号+号
- **模块标记**:
  - 当事人: `原告[:：]`, `被告[:：]`, `第三人[:：]`
  - 诉请: `诉讼请求[：:]`, `诉称[：:]`
  - 答辩: `辩称[：:]`
  - 查明: `经审理查明[，：]`, `本院查明[：:]`
  - 认为: `本院认为[，：]`
  - 判决: `判决如下[：:]`

---

### 3.2 民事裁定书

#### 与判决书的区别
- **适用范围**: 程序性事项（管辖、保全、撤诉、中止等）
- **标题**: "民事裁定书"
- **结构**: 类似判决书，但结果为"裁定如下"
- **上诉期**: 10日（判决书15日）

#### 常见类型
1. **管辖权异议**: `驳回原告×××对本案管辖权提出的异议`
2. **财产保全**: `查封/冻结×××价值×××元的财产`
3. **撤诉**: `准许原告×××撤回起诉`
4. **中止审理**: `本案中止诉讼`
5. **终结审理**: `本案终结诉讼`

#### 关键识别
- `裁定如下[：:]`
- `如不服本裁定[，：]`

[案件由来]

#### 关键识别

- **效力说明**: `签收后即具有法律效力`

```

(××××)×刑初××号

[公诉机关]

[案件由来]
××××人民检察院以×检×诉[××××]××号起诉书指控被告人×××犯××罪，于××××年××月××日向本院提起公诉。

[审理情况]
本院依法组成合议庭，公开（或不公开）开庭审理了本案。

[控辩意见]
公诉机关指控：...
被告人×××辩称：...
辩护人×××提出辩护意见：...

[法院审理]
经审理查明：...
本院认为：...

[判决结果]
判决如下：
一、被告人×××犯××罪，判处...
二、...

[尾部]
如不服本判决，可在接到判决书的第二日起十日内...
```

#### 关键识别
- **标题**: "刑事判决书"
- **案号**: (年份)法院代字+"刑初"+编号+号
- **当事人称谓**:
  - 公诉机关（vs 民事的原告）
  - 被告人（vs 民事的原被告）
  - 辩护人（vs 民事的委托诉讼代理人）
- **模块标记**:
  - 指控: `指控：`, `起诉书指控`
  - 辩护: `辩称：`, `辩护意见`
  - 查明: `经审理查明`
  - 认为: `本院认为`

#### 自然人
```
姓名[，：,]\s*(男|女)[，,]\s*(\d{4})年(\d{1,2})月(\d{1,2})日出生
```

#### 法人/组织
```
名称[：:]\s*([^，。]+公司|[^，。]+局|[^，。]+院)
统一社会信用代码[：:]\s*([A-Z0-9]{18})
住所地[：:]\s*([^。，]+)

```

#### 代理人
```
(委托诉讼代理人|委托代理人|辩护人|法定代理人)[：:]\s*([^，。]+)
```

### 4.3 日期识别规则

```
(\d{4})年(\d{1,2})月(\d{1,2})日
```

### 4.4 金额识别规则

```
(人民币|￥)?\s*(\d{1,3}(,\d{3})*|\d+)\s*元
```

### 5.3 数据结构

```python
@dataclass
class Party:
    """当事人信息"""
    role: str                                   # 身份（原告/被告/申请人等）
    name: str                                   # 姓名/名称
    party_type: str = "natural"                 # 类型：natural/organization
    
    # 自然人
    gender: Optional[str] = None
    birth_date: Optional[str] = None
    nationality: Optional[str] = None
    id_number: Optional[str] = None
    
    # 法人/组织
    unified_code: Optional[str] = None          # 统一社会信用代码
    legal_representative: Optional[str] = None  # 法定代表人
    
    # 通用
    address: Optional[str] = None
    contact: Optional[str] = None
    
    # 代理人
    agent_name: Optional[str] = None
    agent_type: Optional[str] = None
    law_firm: Optional[str] = None

@dataclass
class LegalDocument:
    """法律文书基类"""
    # 基本信息
    document_type: DocumentType
    court_name: str = ""                        # 法院/仲裁委名称
    document_title: str = ""                    # 文书标题
    case_number: str = ""                       # 案号
    case_type: str = ""                         # 案件类型
    
    # 日期
    filing_date: Optional[str] = None           # 立案日期
    hearing_date: Optional[str] = None          # 开庭日期
    decision_date: Optional[str] = None         # 裁判日期
    
    # 当事人
    parties: List[Party] = field(default_factory=list)
    
    # 程序信息
    procedure: str = ""                         # 审理程序
    is_public: bool = True                      # 是否公开审理
    
    # 内容模块
    claims: List[str] = field(default_factory=list)         # 诉讼请求/仲裁请求
    defense: str = ""                                       # 答辩意见
    facts_found: str = ""                                   # 查明事实
    court_opinion: str = ""                                 # 法院/仲裁庭认为
    decision_items: List[str] = field(default_factory=list) # 判决/裁定/裁决主文
    legal_basis: List[str] = field(default_factory=list)    # 法律依据
    
    # 费用
    case_costs: Dict[str, str] = field(default_factory=dict)
    
    # 尾部
    judicial_officers: List[Dict] = field(default_factory=list)  # 审判/仲裁人员
    clerk: str = ""                                         # 书记员
    appeal_rights: str = ""                                 # 上诉权利告知
```

## 六、解析器实现策略

### 6.1 两阶段解析法

```
第一阶段：类型检测
    ↓
检测案号格式
检测标题关键词
检测特有标记
    ↓
确定DocumentType

第二阶段：结构化解析
    ↓
分段：按模块标记分割
    ↓
提取：在各段内提取字段
    ↓
组装：构建LegalDocument对象
```
