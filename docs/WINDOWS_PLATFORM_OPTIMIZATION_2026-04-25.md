# Windows 平台专项优化记录（2026-04-25）

## 已落地优化

### 启动与运行性能

- 主入口延迟导入 `MainWindow`，减少 QApplication 创建前的重型 GUI 模块加载。
- Windows 打包/运行时默认关闭控制台日志输出，避免无控制台 GUI 进程产生额外 IO。
- Windows 下关闭菜单和下拉框动画，降低低配机器上的 UI 响应抖动。
- 新增 `--benchmark-startup` 启动探针，输出启动时间、工作集内存和事件循环 tick 频率。

### 显示效果与 DPI

- QApplication 创建前设置 Windows DPI 策略：
  - Qt 高 DPI 缩放启用。
  - 缩放舍入策略使用 `PassThrough`，减少 125% / 150% 缩放下的边界错位。
  - 优先启用 per-monitor DPI awareness。
- Windows 默认 UI 字体切换为 `Microsoft YaHei UI`，更贴近 Windows 10/11 系统控件。

### 系统兼容性

- 设置 Windows AppUserModelID：`CaseFolderManager.LEXORA.2`，用于任务栏归组和通知识别。
- 预留 Windows 标题栏深色模式入口，当前默认仍保持浅色标题栏，避免与浅色 UI 冲突。

### 用户体验

- 保留并兼容既有 Windows 右键菜单集成。
- 新增 `.lexora-backup` 当前用户级文件关联，双击备份文件可进入导入流程，不需要管理员权限。
- 保留 `Ctrl+N`、`Ctrl+M`、`Ctrl+Q` 等快捷键。
- 新增 Windows 性能采集脚本：`scripts/windows_perf_probe.py`。

## 3. 启动性能检查

### 自动采集命令

```powershell
python scripts\windows_perf_probe.py --runs 5 --json-out reports\windows_perf_after.json
```

如果要做优化前后对比：

```powershell
python scripts\windows_perf_probe.py --runs 5 --json-out reports\windows_perf_before.json
python scripts\windows_perf_probe.py --runs 5 --json-out reports\windows_perf_after.json
```

重点看这些字段：

| 指标 | 含义 | 建议判断 |
|---|---|---|
| `startup_ms.mean` | 平均启动到窗口显示并进入事件循环的时间 | 越低越好；普通电脑建议控制在 3000-5000ms 内 |
| `memory_rss_mb.mean` | 程序窗口启动后的工作集内存 | 越低越好；普通空闲状态建议尽量低于 450MB |
| `peak_working_set_mb.mean` | 启动期间峰值工作集内存 | 观察是否异常飙升 |
| `frame_tick_hz.mean` | 窗口启动后事件循环响应频率近似值 | 接近 60 较理想；明显低于 45 需要排查 |
| `process_elapsed_ms.mean` | 探针进程总耗时 | 用于辅助判断启动和退出是否拖慢 |

记录表：

| 场景 | startup_ms | memory_rss_mb | peak_working_set_mb | frame_tick_hz | 结论 |
|---|---:|---:|---:|---:|---|
| 首次冷启动 |  |  |  |  |  |
| 第二次热启动 |  |  |  |  |  |
| 杀毒软件开启 |  |  |  |  |  |
| 杀毒软件临时关闭 |  |  |  |  |  |
| 打包 EXE 启动 |  |  |  |  |  |

需要进一步优化的信号：

- 冷启动超过 6 秒，且不是杀毒软件首次扫描导致。
- 空闲内存持续高于 600MB。
- 启动期间 CPU 长时间 100%。
- 窗口显示后 2 秒内无法点击。
- 打包 EXE 明显比源码运行慢很多。
