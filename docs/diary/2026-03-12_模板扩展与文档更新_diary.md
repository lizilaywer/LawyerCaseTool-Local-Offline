早上醒来，我像往常一样检查昨天的开发成果。昨天修复了模板管理功能的几个关键bug——新建模板无法保存、删除模板无效、重置功能不完整。这些修复让用户可以真正自由地管理自己的模板库。

## 上午：规划与实现

### 第一步：梳理需求

用户的需求很清晰：

1. **民事案件模板(原告)** - 现有的"民事案件模板"改个名
2. **民事案件模板(被告)** - 全新设计，针对被告方的特殊需求
3. **劳动仲裁模板(申请人)** - 劳动者角度
4. **劳动仲裁模板(被申请人)** - 用人单位角度
5. **商事仲裁模板(申请人)** - 商业争议
6. **商事仲裁模板(被申请人)** - 商业争议

总共8个模板。

还有，**所有模板都要增加"办理阶段"和"联系方式"两个变量**。

### 第二步：开始编码

我打开 `src/config/default_templates.py`，这个文件已经很久没有被大改动了。

首先，把 `CIVIL_TEMPLATE` 重命名为 `CIVIL_PLAINTIFF_TEMPLATE`，改个描述：

```python
CIVIL_PLAINTIFF_TEMPLATE: Dict[str, Any] = {
    "id": "civil_001",
    "name": "民事案件模板(原告)",
    "description": "适用于民事诉讼中原告方的标准化案卷",
    ...
}
```

然后，一个一个地添加新模板。每个模板都要考虑：
- **文件夹结构** - 不同业务场景需要不同的案卷结构
- **变量定义** - 哪些信息是必填的，哪些是选填的
- **默认值** - 给用户一个合理的起点

**民事案件模板(被告)** 的设计最费心思。被告方和原告方有什么不同？
- 被告可能需要提交**管辖权异议申请书**
- 被告可能需要提交**反诉状**
- 被告在执行阶段可能需要提交**执行异议申请书**
- 被告需要知道**原告是谁**

所以我在变量里加了 `plaintiff_name`（原告姓名）。

**劳动仲裁模板**的设计也很讲究。劳动者和用人单位关注的重点完全不同：
- 劳动者关心劳动合同、工资流水、社保记录
- 用人单位关心规章制度、考勤记录、解除合法性

**商事仲裁模板**则更复杂一些。商事争议类型很多：买卖合同、建设工程、股权投资、借款合同、租赁合同...我在"办理阶段"变量里做了细致的区分：

```python
"validation": {
    "options": ["申请", "受理", "组庭", "开庭", "裁决", "执行", "其他"]
}
```

"组庭"这个选项是商事仲裁特有的，劳动仲裁和民事诉讼都没有这个环节。

### 第三步：添加新变量

所有8个模板都要加两个变量：

```python
{
    "key": "case_stage",
    "label": "办理阶段",
    "type": "select",
    "required": False,
    "validation": {...}
},
{
    "key": "contact_info",
    "label": "联系方式",
    "type": "text",
    "required": False,
    "validation": {"max_length": 100}
}
```

这两个变量都是**非必填**的，但都很实用：
- "办理阶段"让律师一眼看清案子进展
- "联系方式"方便随时联系当事人

我为不同模板设计了不同的"办理阶段"选项：
- 民事：一审、二审、再审、执行
- 刑事：侦查、审查起诉、一审、二审、再审、申诉
- 劳动仲裁：申请、受理、开庭、裁决、执行
- 商事仲裁：申请、受理、组庭、开庭、裁决、执行
- 非诉：前期沟通、正式委托、实施阶段、成果交付、结案

细节决定专业度。

## 中午：创建文件夹和文件

代码写完了，但工作还没结束。模板配置文件里指定了新的文件夹路径：
- `templates/civil2/`
- `templates/labor_arbitration/`
- `templates/commercial_arbitration/`

这些文件夹还不存在，我需要创建它们：

```powershell
New-Item -ItemType Directory -Force -Path templates/civil2, templates/labor_arbitration, templates/commercial_arbitration
```

然后，把民事案件的模板文件复制到 civil2 文件夹，这样被告方模板也能使用现有的 Word 模板文件。

## 下午：测试与问题

一切准备就绪，我自信满满地启动软件...

**然后发现问题了。**

主界面左侧还是只显示3个模板：民事案件模板、刑事案件模板、非诉案件模板。

新的模板呢？我明明在 `default_templates.py` 里定义了8个模板啊！

我检查代码，检查配置文件，最后终于明白了：**软件启动时加载的是 `%APPDATA%/LawyerCaseTool/templates.json`，而不是重新加载 `default_templates.py`**。

换句话说，我修改了源代码，但软件用的是用户电脑上的旧配置。

**解决方案**：用户需要点击"重置默认"按钮，或者手动删除配置文件。

我把这个发现告诉了用户。虽然有点尴尬，但至少找到了原因。

## 经验总结

**关于模板设计**：
不要只想着"功能齐全"，要想着"用户场景"。被告方的案卷结构和原告方不一样，劳动者和用人单位关注的重点不一样。好的工具要懂业务。

**关于变量设计**：
"办理阶段"这个变量看似简单，但不同业务场景的选项差异很大。细节之处见真章。

**关于配置管理**：
修改源代码 ≠ 用户能看到变化。要理解软件的运行机制，要知道配置文件在哪里，如何更新。
