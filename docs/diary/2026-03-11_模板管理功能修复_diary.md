## 排查：新建模板保存问题

我打开 `template_manager.py`，仔细检查 `_on_save()` 方法：

```python
def _on_save(self) -> None:
    current_row = self._template_list.currentRow()
    if current_row < 0:
        return

    template = self._editor.get_template()
    template["id"] = self._templates[current_row]["id"]

    self._templates[current_row] = template
    self._template_list.item(current_row).setText(template["name"])

    # 保存到配置
    self._config_manager.update_template(template["id"], template)

    QMessageBox.information(self, "保存成功", "模板已保存")
```

发现问题了！不管是新建模板还是编辑已有模板，都调用 `update_template()`。

但问题是：
- **编辑已有模板**：模板已在配置中，`update_template` 能正常工作
- **新建模板**：模板只在内存列表中，不在配置文件中，`update_template` 会失败

**解决方案**：检查模板是否已存在于配置中：

```python
# 检查模板是否已存在于配置管理器中
existing_template = self._config_manager.get_template(original_id)
if existing_template:
    # 更新已有模板
    self._config_manager.update_template(original_id, template)
else:
    # 新建模板，需要先添加到配置管理器
    self._config_manager.add_template(template)
```

## 排查：删除模板无效

修复完保存问题，用户又反馈：

"删除模板后，重启软件，模板还在..."

我查看 `_on_delete_template()` 方法：

```python
def _on_delete_template(self) -> None:
    current_row = self._template_list.currentRow()
    if current_row < 0:
        return

    reply = QMessageBox.question(...)

    if reply == QMessageBox.StandardButton.Yes:
        del self._templates[current_row]
        self._template_list.takeItem(current_row)
```

**问题很明显**：只从内存列表 `self._templates` 删除，没有从配置文件删除！

**修复方案**：

```python
if reply == QMessageBox.StandardButton.Yes:
    # 从配置管理器中删除
    if self._config_manager.get_template(template_id):
        self._config_manager.delete_template(template_id)
    
    # 从内存列表中删除
    del self._templates[current_row]
    self._template_list.takeItem(current_row)
```

## 优化：重置功能

用户还提出了重置功能的需求：

"默认之前现有的3个模板，每次如果用户点击重置，就重置到初始默认设置。"

原来的重置逻辑：
```python
def _on_reset_templates(self) -> None:
    self._templates = [t.copy() for t in DEFAULT_TEMPLATES]
    self._load_templates()
```

这只会重置内存列表，不会同步到配置文件。

**优化方案**：

```python
def _on_reset_templates(self) -> None:
    # 重置配置管理器中的模板
    self._config_manager.reset_templates()
    
    # 重新加载模板列表
    self._templates = self._config_manager.get_templates()
    self._load_templates()
    
    QMessageBox.information(
        self,
        "重置成功",
        "已恢复到默认模板（民事、刑事、非诉）"
    )
```

同时优化提示信息，明确告知用户重置的影响：
- 删除所有自定义添加的模板
- 恢复民事、刑事、非诉3个默认模板
- 重置对默认模板的所有修改

## 今日总结

| 问题 | 原因 | 修复方案 |
|------|------|----------|
| 新建模板无法保存 | 新建模板不存在于配置中，`update_template` 失败 | 检查存在性，不存在则调用 `add_template` |
| 删除模板无效 | 只从内存删除，未从配置文件删除 | 先调用 `delete_template` 从配置删除 |
| 重置不同步 | 只重置内存列表 | 调用 `reset_templates` 同步到配置 |

**经验总结**：
1. 内存操作和文件操作要同步
2. 新增和更新的逻辑要区分
3. 用户操作后要有明确的反馈
