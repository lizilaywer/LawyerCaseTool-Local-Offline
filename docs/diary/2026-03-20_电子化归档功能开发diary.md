# 2026-03-20 电子化归档功能开发日记

## 开发目标

为律师案卷自动化生成工具新增"电子化归档"功能，实现从现有案卷文件夹中提取信息、生成标准化归档文档。

---

## 需求分析

### 用户场景

律师手头已有整理好的案卷文件夹（包含 Word 文档、PDF、图片等），需要：
1. 浏览现有案卷的文件夹结构
2. 提取文件中的关键信息（当事人姓名、案号、日期等）
3. 生成标准化的归档文档

### 核心功能

1. **选择文件夹** - 打开案卷文件夹
2. **浏览结构** - 显示文件夹树形结构
3. **预览文件** - 支持 Word/PDF/图片预览
4. **提取变量** - 右键菜单提取文字到变量
5. **选择模板** - 从 templates/ 选择归档模板
6. **替换导出** - 变量替换后导出文档

## 遇到的问题与解决

### 问题1：QSizePolicy.DragDropMode.DropOnly 不存在

**错误信息**：
```
AttributeError: 'PySide6.QtWidgets.QSizePolicy' object has no attribute 'DragDropMode'
```

**解决方案**：移除该行代码，只使用 `setAcceptDrops(True)`

### 问题2：QTextCursor 导入位置错误

**错误信息**：
```
ImportError: cannot import name 'QTextCursor' from 'PySide6.QtWidgets'
```

**解决方案**：将 `QTextCursor` 从 `PySide6.QtGui` 导入

### 问题3：.doc 格式文件不支持

**现象**：打开 .doc 文件时报错

**解决方案**：
- python-docx 只支持 .docx 格式
- 添加格式检查，.doc 文件提示用户转换格式

### 问题4：右键菜单变量列表不显示

**原因**：创建预览控件后没有同步变量列表

**解决方案**：在创建 ArchivePreview 后调用 `set_variables()` 方法

### 问题5："设置为变量"功能不生效

**原因**：`_on_variable_set` 方法只更新状态栏，没有实际替换文本

**解决方案**：添加 `_replace_text_in_preview()` 方法实现文本替换

## 经验总结

### 1. UI 组件复用

- 参考现有 `template_maker.py` 的三栏布局
- 复用 `COLORS` 配色方案保持风格一致
- 复用信号连接模式

### 2. 文件预览实现

- Word 预览使用 python-docx 提取文本
- PDF 预览使用 PyMuPDF (fitz)
- 图片预览使用 QPixmap
- 变量高亮使用 QTextCharFormat

### 3. 变量系统设计

- 变量结构：名称 + 变量名 + 值
- 与主界面同步：只增不减策略
- 导入/导出：JSON 格式

### 4. 右键菜单实现

- 使用 `customContextMenuRequested` 信号
- 创建 QMenu 并添加子菜单
- 使用 lambda 传递参数
