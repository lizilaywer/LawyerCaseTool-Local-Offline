# 2026年3月11日 开发日记：让 OCR 识别结果真正"全面"应用

> 一个小改进，让信息识别功能从"能用"变成"好用"

---

## 上午：用户的新需求

昨天刚把 OCR 信息识别功能的界面优化做完，今天用户又提出了一个关键的改进点：

"现在点击'应用到案卷变量'，只有姓名被填充了，其他信息比如性别、出生日期、住址、身份证号都没被应用到主界面..."

我看着截图里的弹窗——"已自动填充 1 个变量到案卷表单"——陷入了沉思。确实，如果只识别出一个字段，那这个功能的价值就大打折扣了。

用户的需求很明确：**所有识别到的字段都应该被应用到案卷变量中去**。如果模板中已有对应的变量，就填充；如果没有，就自动创建。

---

## 中午：分析问题

我先梳理了一下当前的流程：

1. OCR 识别身份证图片 → 得到姓名、性别、出生日期、住址、身份证号等字段
2. 点击"应用到案卷变量"
3. `FieldMatcher.match()` 方法尝试将识别字段匹配到模板变量
4. 只返回匹配成功的字段，其他字段被丢弃

问题就出在第 3、4 步——`match()` 方法只返回已匹配的字段，未匹配的字段被忽略了。

而当前模板中只有一个 `client_name` 变量（委托人姓名）能与"姓名"匹配，其他字段因为没有对应变量而被丢弃。

---

## 下午：实现方案

我决定采用这样的方案：

1. **全面匹配**：不管是否匹配到现有变量，所有识别字段都要返回
2. **自动创建**：未匹配的字段自动创建为新变量
3. **动态添加**：新变量实时添加到表单和模板配置

### 第一步：改造 FieldMatcher

新增 `match_all()` 方法，返回所有识别字段，并标记是"已匹配"还是"新建"：

```python
def match_all(self, recognition_result, template_variables):
    matches = {}
    for source_field, field_conf in recognition_result.fields.items():
        matched_var = self._find_best_match(source_field, ...)
        if matched_var:
            # 已匹配到现有变量
            matches[matched_var] = (value, confidence, 'matched')
        else:
            # 未匹配，创建新变量
            new_var_key = self._field_name_to_var_key(source_field)
            matches[new_var_key] = (value, confidence, 'new')
    return matches
```

同时新增 `_field_name_to_var_key()` 方法，把中文字段名转换为英文变量名（如"身份证号"→"id_number"）。

### 第二步：改造主窗口逻辑

在 `_on_extraction_data_applied()` 方法中：

```python
# 分离已有变量和新变量
for var_key, value in data.items():
    if self._variables_form.has_variable(var_key):
        # 已有变量，直接填充
        existing_values[var_key] = value
    else:
        # 新变量，需要创建
        new_vars_to_add.append(new_var_def)

# 动态添加新变量到表单
for new_var in new_vars_to_add:
    self._variables_form.add_variable(new_var)

# 添加到模板配置
self._add_variables_to_template(new_vars_to_add)
```

### 第三步：改造变量输入控件

新增 `add_variable()` 方法，支持在表单运行时动态添加新的输入控件：

```python
def add_variable(self, var_def):
    input_widget = VariableInput(var_def)
    self._layout.insertWidget(self._layout.count() - 1, input_widget)
    self._inputs[var_def['key']] = input_widget
```

## 傍晚：优化细节

功能跑通后，我又优化了几个细节：

**1. 中文标签映射**

为新变量自动生成友好的中文标签：
```python
def _get_field_label(self, var_key):
    label_mapping = {
        'name': '姓名',
        'gender': '性别',
        'birth_date': '出生日期',
        'address': '住址',
        'id_number': '身份证号',
        # ...
    }
    return label_mapping.get(var_key, var_key)
```

**2. 提示信息优化**

弹窗显示更详细的信息：
```
已自动填充 5 个变量到案卷表单。

✓ 匹配现有变量: 1 个
✓ 创建新变量: 4 个
```

**3. 避免重复创建**

检查变量是否已存在，避免重复添加：
```python
if self._variables_form.has_variable(var_key):
    # 已存在，直接填充
else:
    # 不存在，创建新变量
```
