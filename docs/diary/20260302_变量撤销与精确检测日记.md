## 上午：变量撤销功能

### 需求背景

用户反馈：已经替换为变量的文字（如 `{{client_name}}`），如果替换错了，想要撤销回原来的文字，目前只能点击工具栏的"撤销"按钮——但那会撤销所有修改。

用户希望：
1. 右键点击变量，可以选择"撤销此变量"（只撤销这一个）
2. 或者"撤销所有同名变量"（撤销文档中所有 `{{client_name}}`）

### 实现思路

**1. WordEditor 层面**

需要在历史记录中存储"变量 → 原始文字"的映射：

```python
def undo_variable(self, variable_name: str, single: bool = False) -> int:
    # 从历史中找到原始文字
    original_text = self.get_variable_original_text(variable_name)

    # 执行反向替换：{{xxx}} → 原始文字
    count = self.replace_text(f"{{{{{variable_name}}}}}", original_text, not single)

    return count
```

**2. 预览控件层面**

检测鼠标位置是否在变量上，显示对应的右键菜单：

```python
# 新增信号
undo_variable_requested = Signal(str, bool)  # 变量名, 是否单个撤销

# 检测变量
variable_name = self._detect_variable_at_position(pos)

if variable_name:
    menu.addAction("撤销此变量")
    menu.addAction("撤销所有 {{xxx}}")
```

**3. 主窗口层面**

连接信号，调用 WordEditor 的撤销方法。

## 下午：变量检测精度问题

### 问题暴露

功能上线后测试发现：**点击变量旁边的普通文字，也会显示撤销菜单！**

### 问题分析

原来的检测逻辑：
```python
# 向前搜索 {{
start = current_pos
while start > current_pos - 100:
    if full_text[start:start+2] == '{{':
        break
    start -= 1

# 向后搜索 }}
end = current_pos
while end < current_pos + 100:
    if full_text[end:end+2] == '}}':
        break
    end += 1
```

**问题**：只要当前位置前后 100 字符内有 `{{` 和 `}}`，就认为在变量上。

比如文档内容是：
```
你好{{client_name}}，欢迎
```
当点击"你好"时：
- 向前搜索 100 字符 → 找不到 `{{`（start 停在当前位置）
- 向后搜索 100 字符 → 找到了 `}}`
- 但由于逻辑问题，仍可能误判

### 正确的解决方案

**只有当光标位置确实在某个 `{{xxx}}` 变量的范围内时才判定为变量。**

```python
def _detect_variable_at_position(self, pos) -> Optional[str]:
    cursor = self._text_edit.cursorForPosition(pos)
    current_pos = cursor.position()
    full_text = self._text_edit.toPlainText()

    # 使用正则找到所有变量及其精确位置
    var_pattern = re.compile(r'\{\{([^}]+)\}\}')

    for match in var_pattern.finditer(full_text):
        var_start = match.start()
        var_end = match.end()

        # 检查光标是否在这个变量的范围内
        if var_start <= current_pos <= var_end:
            return match.group(1).strip()

    return None
```

**关键改进**：
1. 使用 `finditer` 遍历所有变量，获取精确的 `start` 和 `end` 位置
2. 只有 `var_start <= current_pos <= var_end` 时才认为在变量上
3. 这样就不会把变量旁边的文字误判为变量

### 经验教训

**"范围检测" vs "搜索检测"**

- **搜索检测**：从当前位置向前后搜索标记 → 容易误判
- **范围检测**：先找到所有目标，检查当前位置是否在其中 → 精确可靠

这就像找座位：
- 错误方式：从你站着的地方向前后各走100步，找到座位就认为你坐着
- 正确方式：先列出所有座位的范围，检查你是否在某个座位的范围内
