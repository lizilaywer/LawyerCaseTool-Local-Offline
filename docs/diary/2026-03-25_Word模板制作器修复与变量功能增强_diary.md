### 1. Word模板制作器跨Run文本替换修复

**问题发现**：用户在群里反馈说，在Word模板制作器中选中"安徽省仁盛实业有限公司"这样的文字，无法替换为变量。一开始我还以为是选中文本的问题，结果一看代码，发现问题没那么简单。

**深入研究**：为了搞清楚问题根源，我写了个测试脚本，用python-docx读取文档，打印出每个段落的Run结构。结果让我大吃一惊——原来Word文档里的文本并不是连续存储的！一个看起来连贯的"安徽省仁盛实业有限公司"，在Word内部可能被分割成多个Run对象：

```
Run 1: "安徽省"
Run 2: "仁盛"
Run 3: "实业"  
Run 4: "有限公司"
```

**恍然大悟**：这就是问题所在！原来的替换逻辑只在单个Run内检查`if old_text in run.text`，如果文本跨多个Run，自然就匹配不到了。

**解决方案**：重写`_replace_in_paragraph`方法，核心思路是：

1. 先在段落级别检查文本是否存在（`paragraph.text`）
2. 构建Run映射表，记录每个字符位置属于哪个Run
3. 找到匹配文本跨越的Run范围
4. 对于跨Run的情况，采用"保留首Run、清空中间Run、截取末Run"的策略
5. 从后向前处理多个匹配位置，避免位置偏移问题

**测试验证**：修复后测试了各种场景：
- ✅ 跨Run的公司名称替换
- ✅ 包含空格的文本替换  
- ✅ 单Run内的简单替换（向后兼容）

**心得体会**：这个bug让我深刻理解了Word文档的内部结构。有时候表面上的问题，根源在更深层次。花时间去理解底层原理，虽然耗时，但值得。

### 2. 添加自定义变量功能增强

**用户需求**：用户希望Word模板制作器中的"+ 添加自定义变量"功能，能和模板管理器中的"变量定义"保持一致，最好还能选择归属哪个模板。

**设计思路**：既然要统一体验，那就干脆做得彻底一些：

1. 新建一个`AddVariableDialog`类，界面和模板管理器保持一致
2. 支持完整的字段：键名、标签、类型、必填、归属模板
3. 归属模板下拉框要显示所有可用模板，按分类组织
4. 添加的变量要自动保存到对应模板的配置中

**实现细节**：
- 键名验证：只允许英文字母、数字、下划线，且不能以数字开头
- 模板列表：从ConfigManager获取用户模板，如果没有则使用DEFAULT_TEMPLATES
- 显示格式：`[分类] 模板名`，如`[民事简易] 民事案件简易模板(原告)`
- 自动保存：调用`config_manager.update_template()`将变量写入配置

**意外插曲**：在测试时发现，添加变量后窗口会重复弹出。排查后发现是下拉框的`currentIndexChanged`信号问题——添加变量后下拉框仍然选中"+ 添加自定义变量..."选项，导致信号再次触发。

**解决方法**：在打开添加变量对话框**之前**，立即将下拉框重置为默认选项（"-- 选择变量 --"），这样无论对话框结果如何，都不会重复触发。

**代码示例**：
```python
def _on_variable_changed(self) -> None:
    if self._var_combo.currentData() == "__custom__":
        # 立即重置下拉框选择，防止重复触发
        self._var_combo.setCurrentIndex(0)
        # ... 打开对话框
```

### 3. blank.docx空文件问题修复

**问题发现**：用户生成案卷时报错：`Package not found at 'templates\blank.docx'`

**排查过程**：一看文件确实存在，但大小为0字节！原来这是一个空文件，不是有效的Word文档。

**快速修复**：用python-docx重新创建了一个有效的空白文档：
```python
from docx import Document
doc = Document()
doc.save('templates/blank.docx')
```

**反思**：这种低级错误不应该发生。以后创建占位文件时，一定要确保文件内容是有效的。

## 今日技术收获

### 1. python-docx的Run机制

Word文档的段落（Paragraph）由多个Run组成，每个Run可以有不同的格式（字体、颜色、加粗等）。当用户在Word中编辑文本时，Word会根据格式变化自动分割Run。

**应用场景**：
- 文本替换时必须考虑跨Run的情况
- 保持格式时需要谨慎处理Run的合并和分割
- 从后向前处理可以避免位置偏移问题

### 2. PySide6信号处理经验

**重复触发问题**：下拉框的`currentIndexChanged`信号在选择项变化时触发。如果在处理函数中改变选择项，可能导致连锁反应。

**最佳实践**：
- 在处理特殊选项（如"+ 添加..."）时，先重置选择项再执行操作
- 使用`blockSignals()`方法可以临时阻断信号，但要记得恢复
- 对于复杂的交互流程，考虑使用模态对话框阻塞用户操作

### 3. 配置管理设计

ConfigManager的单例模式设计得很巧妙，通过`get_config_manager()`获取实例，确保全局配置的一致性。

**经验总结**：
- 用户配置和默认模板要分开管理
- 修改配置后要立即保存，避免数据丢失
- 提供配置验证和默认值回退机制
