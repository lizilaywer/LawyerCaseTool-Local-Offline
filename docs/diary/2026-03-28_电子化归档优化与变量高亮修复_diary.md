### 1. 那个让人抓狂的高亮问题

用户反馈说在电子化归档界面，选中文字右键"设置为变量"后，变量没有高亮显示。我一看截图，确实——`{{case_number}}`、`{{matter_number}}` 这些变量就像普通文字一样躺在那里，毫无存在感。

**问题根源**: 原来的代码只在打开文档时做一次高亮，设置变量后只发送了信号，没有更新UI。

**我的修复思路**: 
- 先替换选中的文本为变量占位符
- 然后立即重新高亮所有变量
- 最后才发送信号通知其他组件

看似简单的逻辑，但要在 QTextEdit 里正确处理光标位置、保留文本格式，还是需要小心处理。好在我之前写过类似的高亮逻辑，直接复用了 `_highlight_text_with_variables()` 方法。

**代码片段**:
```python
def _on_set_variable(self, var_key: str, text: str) -> None:
    cursor = self._text_preview.textCursor()
    cursor.insertText(f"{{{{{var_key}}}}}")  # 替换
    self._original_text = self._text_preview.toPlainText()
    self._highlight_text_with_variables(self._original_text)  # 重新高亮
    self.variable_set.emit(var_key, text)
```

### 2. 变量面板的脱胎换骨

用户还提了个需求：左侧变量面板要支持拖拽排序，还要卡片化设计。

**拖拽排序的实现**

这是今天最耗时的部分。我一开始想用 QPushButton 做拖拽手柄，结果事件被拦截得一塌糊涂。后来改用 QLabel 才搞定。

关键逻辑:
1. 在 `mousePressEvent` 记录起始位置
2. 在 `mouseMoveEvent` 判断是否满足拖拽条件（移动距离 > 15px）
3. 创建 `QDrag` 对象，设置半透明缩略图
4. 在父容器的 `dropEvent` 里处理重新排序

**卡片化设计**

每个变量项现在是一个圆角卡片，有阴影、有悬停效果。左侧加了个 "≡" 符号作为拖拽提示。整体看起来现代多了。

**踩坑记录**:
- 拖拽时卡片要半透明，但不能太透明，否则看不清内容
- 放置指示器的位置计算要精确，不然用户体验很差
- 要处理好stretch spacer，不然布局会乱

## 开发心得

### 关于拖拽排序

Qt 的拖拽系统其实挺强大的，但文档写得不怎么友好。我花了好长时间才搞清楚 `dragEnterEvent`、`dragMoveEvent`、`dropEvent` 这几个事件的触发时机和传递机制。

一个小技巧：在 `dragMoveEvent` 里实时计算放置位置并显示指示器，这样用户体验会好很多。不要等到了 `dropEvent` 才处理。

### 关于 QTextEdit 的高亮

`QTextEdit` 的高亮有两种做法：
1. 用 `QSyntaxHighlighter`——适合代码编辑器，按行处理
2. 用 `QTextCursor` 插入带格式的文本——适合这种变量高亮

我选的是第二种，因为变量可能跨行，而且需要精确控制格式。

### 关于用户体验

今天深刻体会到，细节决定成败。一个小小的拖拽手柄、一个放置指示器、一个悬停效果，都能让用户体验提升一个档次。

但也别过度设计。我一开始想加动画效果，后来想想还是算了，简单直接最好。
