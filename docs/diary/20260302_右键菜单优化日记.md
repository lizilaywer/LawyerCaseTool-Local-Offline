# 2026-03-02 开发日记：## 一个"小"功能的"大"学问

今天只做了一件事，但这件事让我对 Qt 的控件机制有了更深的理解。

### 上午：用户的"小"需求

用户说想在 Word 模板制作器的预览区域添加右键菜单功能——选中文字后右键，直接弹出"替换选中"对话框，而不是每次都要点击工具栏的"替换选中"按钮。

听起来很简单对吧？我也这么觉得。于是我信心满满地写下了：

```python
def contextMenuEvent(self, event):
    menu = QMenu(self)
    # ...添加菜单项
    menu.exec(event.globalPos())
```

然后测试——**没有效果**。右键菜单还是原来的 Copy 和 Select All。

### 中午：错误的"位置"

我愣了一下，代码明明写对了啊？`contextMenuEvent` 是 QWidget 的虚方法，重写它应该就能拦截右键菜单...

等等，我突然意识到问题所在：

**`contextMenuEvent` 是针对 `WordPreviewWidget` 这个父控件的，但用户实际右键点击的是里面的 `_text_edit`（QTextEdit）子控件！**

这就像你给房子的门装了门铃，但客人走的却是窗户——门铃当然不会响。

### 下午：正确的解决方案

找到问题后，解决方案就清晰了。需要让 `_text_edit` 自己"放弃"默认菜单，改用我们自定义的：

```python
# 1. 告诉 QTextEdit："不要用默认菜单了，我自己来"
self._text_edit.setContextMenuPolicy(Qt.ContextMenuPolicy.CustomContextMenu)

# 2. 连接信号："当用户右键点击时，调用我的方法"
self._text_edit.customContextMenuRequested.connect(self._show_context_menu)
```

然后实现 `_show_context_menu` 方法：

```python
def _show_context_menu(self, pos):
    menu = QMenu(self)
    cursor = self._text_edit.textCursor()

    if cursor.hasSelection():
        replace_action = menu.addAction("替换选中...")
        replace_action.triggered.connect(self._on_replace_action)
    else:
        no_select = menu.addAction("请先选择文字")
        no_select.setEnabled(False)

    global_pos = self._text_edit.mapToGlobal(pos)
    menu.exec(global_pos)
```

### 晚上：经验总结

这个"小"功能让我学到了一个重要教训：

**在 Qt 中，事件处理是"谁收到谁处理"的。**

- `contextMenuEvent` 只会对控件自己生效
- 如果用户交互发生在子控件上，需要在子控件上设置事件处理

**正确的姿势：**
1. 如果要给整个窗口添加菜单 → 重写 `contextMenuEvent`
2. 如果要给特定子控件添加菜单 → 使用 `setContextMenuPolicy(CustomContextMenu)` + `customContextMenuRequested` 信号

这就像你不能指望用遥控器控制邻居家的电视——得先确保信号发对了地方。
