# 2026年3月11日 开发日记：默认模板大扩展

> 从3个模板到8个模板，覆盖更多业务场景

---

## 背景

今天收到用户的新需求：

1. 将"民事案件模板"改名为"民事案件模板(原告)"
2. 新增"民事案件模板(被告)"
3. 新增劳动仲裁模板（申请人、被申请人）
4. 新增商事仲裁模板（申请人、被申请人）
5. 所有模板增加"办理阶段"和"联系方式"变量

这是一个大工程，需要修改配置文件、创建新文件夹、复制模板文件。

---

## 第一步：修改 default_templates.py

### 1.1 重命名现有模板

将 `CIVIL_TEMPLATE` 重命名为 `CIVIL_PLAINTIFF_TEMPLATE`，更新名称和描述：

```python
CIVIL_PLAINTIFF_TEMPLATE: Dict[str, Any] = {
    "id": "civil_001",
    "name": "民事案件模板(原告)",
    "description": "适用于民事诉讼中原告方的标准化案卷",
    ...
}
```

### 1.2 新增民事案件模板(被告)

创建 `CIVIL_DEFENDANT_TEMPLATE`，与原告模板类似，但针对被告方优化：
- 增加"管辖权异议申请书"
- 增加"反诉状"
- 增加"执行异议申请书"
- 变量中添加"原告姓名"字段

```python
CIVIL_DEFENDANT_TEMPLATE: Dict[str, Any] = {
    "id": "civil_002",
    "name": "民事案件模板(被告)",
    "category": "civil2",
    "template_file": "templates/civil2/template.docx",
    ...
}
```

### 1.3 新增劳动仲裁模板

创建两个劳动仲裁模板：

**申请人模板** (`LABOR_ARBITRATION_APPLICANT_TEMPLATE`)：
- 侧重劳动者权益保护
- 包含劳动合同、工资流水、社保记录等证据材料
- 支持执行申请材料

**被申请人模板** (`LABOR_ARBITRATION_RESPONDENT_TEMPLATE`)：
- 侧重用人单位应诉
- 包含规章制度、考勤记录等
- 支持反申请

### 1.4 新增商事仲裁模板

创建两个商事仲裁模板：

**申请人模板** (`COMMERCIAL_ARBITRATION_APPLICANT_TEMPLATE`)：
- 支持多种争议类型（买卖合同、建设工程、股权投资等）
- 包含仲裁协议、合同文本、往来函件等
- 支持财产保全申请

**被申请人模板** (`COMMERCIAL_ARBITRATION_RESPONDENT_TEMPLATE`)：
- 包含管辖权异议、反请求等
- 支持执行异议

### 1.5 添加新变量

所有模板新增两个变量：

```python
{
    "key": "case_stage",
    "label": "办理阶段",
    "type": "select",
    "required": False,
    "validation": {
        "options": ["一审", "二审", "再审", "执行", "其他"]
    }
},
{
    "key": "contact_info",
    "label": "联系方式",
    "type": "text",
    "required": False,
    "validation": {"max_length": 100}
}
```

不同模板的"办理阶段"选项根据业务场景定制：
- 民事：一审、二审、再审、执行
- 刑事：侦查、审查起诉、一审、二审、再审、申诉
- 劳动仲裁：申请、受理、开庭、裁决、执行
- 商事仲裁：申请、受理、组庭、开庭、裁决、执行
- 非诉：前期沟通、正式委托、实施阶段、成果交付、结案
