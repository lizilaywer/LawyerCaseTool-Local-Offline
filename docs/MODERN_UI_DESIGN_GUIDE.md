# 律师案卷工具 v2.0 - 现代化UI设计方案

## 设计概述

本文档详细说明律师案卷工具的全新UI设计方案，参考了2025年主流应用的设计语言。

## 设计参考

### 参考应用

| 应用名称 | 设计特点 | 借鉴元素 |
|----------|----------|----------|
| **VS Code** | 专业工具、侧边栏布局、 活动栏 |
| **Linear** | 现代项目管理 | 简洁的卡片设计, 标签分类 |
| **Notion** | 知识管理 | 块编辑器、 实时预览 |
| **GitHub Desktop** | 开发工具 | 三栏布局, 状态反馈 |
| **Figma** | 设计工具 | 工具栏设计 | 图层面板 |

### 设计原则

1. **清晰的信息层次** - 重要信息优先显示
2. **一致的视觉语言** - 统一的颜色、 间距、 圆角
3. **高效的交互路径** - 减少点击次数
4. **即时反馈** - 操作状态清晰可见
5. **可预测性** - 符合用户心智模型

## 核心设计元素

### 1. 布局结构

```
┌─────────────────────────────────────────────────────────────┐
│  顶部工具栏 (面包屑导航 + 全局操作)                    │
├─────────────────────────────────────────────────────────────┤
│        │                    │                    │        │
│  左侧  │      中间        │      右侧      │
│  模板  │   变量表单      │   预览     │
│  导航  │   输入         │   面板     │
│  260px│   自适应       │   320px    │
│        │                    │                    │        │
└─────────────────────────────────────────────────────────────┘
```

**类比**：就像一个专业的厨房，- 左侧是食材区库 (模板导航)
- 中间是料理台 (变量输入)
- 右侧是出菜口 (预览结果)

### 2. 配色系统

#### 主色调
```css
/* 专业蓝 - 值得信赖的专业感 */
--primary: #2563eb;
--primary-hover: #1d4ed8;
--primary-light: #dbeafe;
```

#### 语义色
```css
/* 成功 - 绿色 */
--success: #10b981;

/* 警告 - 黄色 */
--warning: #f59e0b;

/* 危险 - 红色 */
--danger: #ef4444;

/* 信息 - 蓝色 */
--info: #3b82f6;
```

#### 中性色
```css
/* 背景层次 */
--gray-50: #f9fafb;  /* 最浅背景 */
--gray-100: #f3f4f6;  /* 卡片背景 */
--gray-200: #e5e7eb;  /* 边框 */
--gray-300: #d1d5db;  /* 禁用边框 */

/* 文字层次 */
--gray-400: #9ca3af;  /* 占位符 */
--gray-500: #6b7280;  /* 次要文字 */
--gray-600: #4b5563;  /* 正文 */
--gray-700: #374151;  /* 标题 */
--gray-800: #1f2937;  /* 强调标题 */
```

**类比**：颜色就像装修风格
- 主色调是墙面主色 (专业蓝)
- 语义色是功能房间标记 (成功绿、警告黄)
- 中性色是家具和装饰 (灰度系统)

### 3. 字体系统

```css
/* 字体家族 */
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;

/* 字号层次 */
12px - 最小文字 (标签、提示)
13px - 正文文字 (表单、列表)
14px - 按钮文字
16px - 小标题
18px - 大标题
```

### 4. 间距系统

```css
/* 基础间距单位 */
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 20px;
--space-6: 24px;
```

**类比**：间距就像呼吸空间
- 太挤会让人窒息
- 太散会让人迷茫
- 恰当的间距让界面"呼吸顺畅"

### 5. 圆角系统

```css
/* 圆角层次 */
--radius-sm: 4px;   /* 小元素 (标签) */
--radius-md: 8px;   /* 中元素 (按钮) */
--radius-lg: 12px;  /* 大元素 (卡片) */
--radius-xl: 16px;  /* 容器 (面板) */
```

## 组件设计

### 1. 模板卡片

**结构**：
```
┌─────────────────────────────┐
│ 🏷️ 模板名称            │
│ 模板描述文字...        │
│ 📁 8个文件夹 · 10个文件 │
└─────────────────────────────┘
```

**交互状态**：
- 默认：浅灰背景，- 悬停：边框高亮
- 选中：蓝色背景 + 左侧指示条

**类比**：像一本书的封面
- 一眼就能看出是什么类型的书
- 选中的书会有书签标记

### 2. 变量表单

**分组设计**：
```
基本信息组 (必填)
├── 案号
├── 当事人姓名
├── 身份证号
└── 联系电话

案件信息组 (选填)
├── 办理阶段
├── 案件类型
├── 备注
└── ...
```

**类比**：像填表
- 必填项是红色星号 (*)
- 选填项可以跳过
- 相关信息分组在一起

### 3. 预览面板

**层级结构**：
```
📁 根目录 (2024-001_张三)
├── 📁 1委托手续
│   ├── 📄 委托合同.docx 📎
│   └── 📄 授权委托书.docx
├── 📁 2证据材料
└── ...
```

**视觉标记**：
- 文件夹：黄色图标
- 模板文件: 蓝色标签 📎
- 普通文件: 白色图标

**类比**：像文件资源管理器的树形视图
- 一目了然的结构
- 清晰的层级关系

### 4. 工具栏

**功能分区**：
```
[路径选择] [清空] [OCR] | [生成]
```

**视觉层次**：
- 次要操作：灰色按钮
- 主要操作：蓝色按钮
- 危险操作：红色按钮

**类比**：像厨房的工具架
- 常用工具伸手可及
- 分门别类摆放
- 常用工具更显眼

## 交互设计

### 1. 模板选择流程

```
用户点击模板 → 高亮选中 → 加载变量定义 → 更新预览
```

**反馈**：
- 点击时有视觉反馈 (高亮)
- 选中后左侧显示指示条
- 变量表单自动更新

### 2. 变量输入流程

```
用户输入 → 宽时验证 → 实时更新预览
```

**验证规则**：
- 必填项：不能为空
- 格式验证: 身份证18位、手机11位
- 长度限制: 最大/最小字符数

### 3. 生成流程

```
点击生成 → 磨贴确认 → 选择路径 → 开始生成 → 进度显示 → 完成提示
```

**进度反馈**：
- 进度条显示当前进度
- 文字说明当前操作
- 可随时取消

## 噪音与动画

### 1. 过渡动画

```css
/* 统一过渡 */
transition: all 0.2s ease;
```

**应用场景**：
- 悬停状态变化
- 选中状态变化
- 显示/隐藏元素

### 2. 加载动画

**骨架屏加载**：
```
┌─────────────┐
│ ████████████ │  ← 灰色闪烁
│ ████████████ │
└─────────────┘
```

**类比**：像建筑工地的临时围挡
- 在真正的内容出现前显示
- 给用户"正在加载"的预期

### 3. 成功动画

**打勾动画**：
```
✓  ← 弹出并消失
```

**类比**：像快递送达的确认
- 短暂但明确的反馈
- 告诉用户"操作成功"

## 可访问性

### 键盘支持

- Tab: 在模板间切换
- Enter: 确认生成
- Esc: 取消/关闭
- Ctrl+S: 保存

### 颜色对比

- 文字与背景对比度 ≥ 4.5:1
- 焦点状态有明显边框
- 错误信息使用红色
