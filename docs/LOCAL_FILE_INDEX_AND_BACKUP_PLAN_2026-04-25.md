# 本地文件级索引与非加密备份迁移实现计划（2026-04-25）

## 目标范围

本计划用于下次开发前确认。确认后即可按本文分阶段实现。

当前只实现两个方向：

1. 文件及证据索引第一阶段：文件级索引与检索。
2. 设置页新增本地备份与一键迁移：先做非加密版本，加密能力暂不实现。

当前明确不做：

- 不做全文索引。
- 不做 PDF/Word/图片 OCR 内容索引。
- 不做云端能力。
- 不做大模型能力。
- 不做加密压缩备份。

设计约束：

- 完全本地离线运行。
- 普通 macOS / Windows 电脑可流畅使用。
- 不强制扫描用户未导入的软件外部目录。
- 大目录和大文件操作必须后台执行，不阻塞 UI。
- 所有新功能应可关闭、可重建、可恢复。
- 文件搜索不新增独立搜索框，必须复用顶部现有全局搜索框，并在搜索框右侧增加“案件/文件”模式切换按钮。
- 备份与迁移功能必须放在“设置”里，不在工作台或其他页面新增主入口。

## 功能 1：文件级索引与检索

### 1.1 实现目标

对已导入案件的文件夹建立本地文件级索引，让用户可以快速搜索：

- 案件名称
- 文件名
- 文件扩展名
- 文件所在目录
- 文件路径
- 文件大小
- 文件修改时间
- 文件是否仍存在

最终效果：

- 在案件中心或工作台提供“本地文件搜索”入口。
- 输入关键词后可快速列出命中文件。
- 点击搜索结果可打开文件或定位到案件目录。
- 索引完成后搜索应接近秒搜。

### 1.2 存储方案

使用 SQLite 数据库，保存到应用数据目录，例如：

```text
<app_data_dir>/search/file_index.sqlite3
```

建议新增模块：

```text
src/core/search/
├── __init__.py
├── file_indexer.py
├── file_index_store.py
├── file_search_service.py
└── models.py
```

建议新增数据库表：

```sql
cases_indexed
- case_id TEXT PRIMARY KEY
- case_name TEXT
- case_path TEXT
- indexed_at TEXT
- last_scan_started_at TEXT
- last_scan_finished_at TEXT
- status TEXT

file_entries
- id TEXT PRIMARY KEY
- case_id TEXT
- case_name TEXT
- root_path TEXT
- relative_path TEXT
- absolute_path TEXT
- parent_relative_path TEXT
- filename TEXT
- stem TEXT
- extension TEXT
- file_type TEXT
- size_bytes INTEGER
- modified_at REAL
- exists INTEGER
- indexed_at TEXT

index_meta
- key TEXT PRIMARY KEY
- value TEXT
```

索引策略：

- `id` 可使用 `case_id + relative_path` 的稳定 hash。
- 使用普通 SQLite 索引加速搜索。
- 第一阶段先不使用 FTS5，避免中文分词和全文检索复杂度。
- 搜索使用 `filename LIKE`、`relative_path LIKE`、`case_name LIKE`，配合必要字段索引。

### 1.3 扫描策略

默认只扫描已导入案件的根目录。

扫描规则：

- 跳过隐藏目录：`.git`、`.DS_Store`、`__pycache__`、系统隐藏文件。
- 跳过临时文件：`~$*.docx`、`.tmp`、`.crdownload` 等。
- 记录文件，不读取文件内容。
- 大目录扫描必须后台执行。
- 扫描过程支持取消。
- 扫描结束后清理已删除文件的索引记录，或标记为 `exists=0`。

建议文件类型分类：

```text
pdf       .pdf
word      .doc .docx
image     .jpg .jpeg .png .bmp .gif .tif .tiff
text      .txt .md .csv
sheet     .xls .xlsx
archive   .zip .rar .7z
other     其他
```

### 1.4 增量更新

第一版实现简单可靠的增量：

- 对比 `relative_path`、`size_bytes`、`modified_at`。
- 文件未变化则不重复更新。
- 新文件插入。
- 已删除文件标记为不存在。

暂不实现实时文件监听。

后续可选增强：

- 定时轻量扫描。
- 用户手动“重新索引当前案件”。
- 启动后空闲时后台维护。

### 1.5 UI 入口

第一版不新增独立搜索框，不做复杂新页面。

固定入口方案：

1. 复用顶部现有全局搜索框。
2. 在搜索框右侧增加一个紧凑切换按钮，位置为当前顶部栏搜索框右侧空白区域。
3. 切换按钮用于在“案件搜索”和“文件搜索”之间切换。
4. 默认模式保持“案件搜索”，避免改变用户当前习惯。
5. 切换为“文件搜索”后，搜索框 placeholder 改为“搜索文件...”。
6. 回车或输入触发搜索时，根据当前模式调用不同搜索服务。
7. 不再额外增加第二个搜索框。

切换按钮建议：

- 可使用分段按钮或小型胶囊按钮。
- 显示文本可为“案件 / 文件”或仅显示当前模式“案件”“文件”。
- 切换后应有明确选中态。
- 宽度控制在顶部栏内，不挤压“新建案件”按钮。

文件搜索结果展示：

- 第一版可复用案件搜索跳转逻辑，但结果弹窗/面板需要区分文件结果。
- 如果当前模式为“文件搜索”，回车后进入文件搜索结果视图或弹出结果面板。
- 搜索结果不要混入案件结果，避免用户误解。

搜索结果展示字段：

- 文件名
- 案件名
- 所在目录
- 文件类型
- 修改时间
- 文件大小

结果操作：

- 打开文件。
- 打开所在文件夹。
- 复制路径。
- 如果文件不存在，显示“文件已丢失，建议重新索引”。

### 1.6 后台任务与状态

建议新增通用索引任务：

```text
FileIndexWorker
- progress_changed(int, str)
- finished(summary)
- failed(error)
- canceled()
```

状态提示：

- 正在扫描案件目录。
- 已扫描 N 个文件。
- 已完成：新增 N，更新 N，失效 N。
- 扫描失败文件清单。

### 1.7 设置项

设置页建议增加：

- 启用文件级索引：开关，默认开启。
- 自动维护索引：开关，默认关闭或轻量开启。
- 重建全部索引。
- 清空索引。
- 显示索引数据库大小。

### 1.8 测试计划

新增测试文件：

```text
tests/test_file_index_store.py
tests/test_file_indexer.py
tests/test_file_search_service.py
```

测试覆盖：

- 首次扫描插入文件记录。
- 修改文件后增量更新。
- 删除文件后标记失效。
- 搜索文件名。
- 搜索目录片段。
- 按案件限制搜索。
- 跳过隐藏目录和临时文件。

### 1.9 验收标准

功能验收：

- 导入案件后可为案件建立文件级索引。
- 可以按文件名/路径/案件名搜索。
- 搜索结果可打开文件和所在目录。
- 删除或移动文件后，重新索引能反映最新状态。
- 大目录扫描时 UI 不冻结。

性能验收：

- 1,000 个文件以内扫描体验流畅。
- 10,000 个文件以内后台扫描不阻塞 UI。
- 索引完成后关键词搜索响应应在可感知的瞬间完成。

## 功能 2：本地非加密备份与一键迁移

### 2.1 实现目标

在设置中增加“备份与迁移”功能，支持：

- 导出软件数据备份包。
- 导入备份包恢复数据。
- 可选是否包含真实案件文件夹。
- 暂不加密。

第一版重点是稳定、透明、可检查。

### 2.2 备份内容

默认备份：

- 应用配置：`config.json`
- 模板配置：`templates.json`
- 案件台账：`cases.json`
- 常用标签、筛选记忆等本地配置
- 自定义模板文件
- sidecar 元数据
- 版本信息和备份清单

可选备份：

- 已导入案件的真实文件夹、子文件夹、文件
- OCR 缓存
- 文件级索引数据库

建议第一版默认不包含真实案件文件夹，避免备份包过大。

### 2.3 备份包格式

建议使用普通 zip 或 tar 格式，扩展名：

```text
.lexora-backup
```

内部结构：

```text
manifest.json
checksums.json
app_data/
  config.json
  templates.json
  cases.json
templates/
sidecars/
case_files/          可选
index/               可选
logs/backup.log      可选
```

`manifest.json` 建议字段：

```json
{
  "format": "lexora-backup",
  "format_version": 1,
  "app_name": "LEXORA",
  "app_version": "2.0.0",
  "created_at": "2026-04-25T00:00:00",
  "platform": "macos/windows",
  "include_case_files": false,
  "include_index": false,
  "include_ocr_cache": false,
  "case_count": 0,
  "file_count": 0,
  "total_size_bytes": 0
}
```

`checksums.json`：

- 使用 SHA-256。
- 用于导入前校验备份包完整性。

### 2.4 新增模块

建议新增：

```text
src/core/backup/
├── __init__.py
├── backup_manifest.py
├── backup_service.py
├── backup_writer.py
├── backup_reader.py
└── restore_service.py
```

职责：

- `backup_manifest.py`：备份清单数据结构。
- `backup_writer.py`：创建备份包。
- `backup_reader.py`：读取和校验备份包。
- `backup_service.py`：面向 UI 的备份入口。
- `restore_service.py`：导入和恢复。

### 2.5 UI 设计

备份与迁移功能固定放在设置页，不在工作台、工具中心、案件中心增加主入口。

设置页新增“备份与迁移”分组。

备份区域：

- 备份软件配置与台账。
- 包含自定义模板。
- 包含 sidecar 元数据。
- 可选：包含真实案件文件夹。
- 可选：包含索引数据库。
- 可选：包含 OCR 缓存。
- 选择备份保存位置。
- 开始备份。

导入区域：

- 选择 `.lexora-backup` 文件。
- 读取备份清单。
- 展示备份创建时间、版本、平台、案件数、是否包含案件文件。
- 选择导入策略：
  - 仅导入配置/模板/台账。
  - 导入并合并到当前数据。
  - 覆盖当前数据（危险，需要二次确认）。
- 如果包含案件文件夹，选择恢复目标目录。

### 2.6 导入策略

第一版建议支持两个策略：

1. 合并导入

- 不删除当前数据。
- 导入新模板、新案件。
- 如果案件 ID 冲突，保留当前记录，给导入记录生成新 ID 或提示用户选择。

2. 覆盖导入

- 导入前先自动创建当前数据备份。
- 覆盖配置、模板、台账。
- 出错时尽量回滚。

第一版推荐默认使用“合并导入”。

### 2.7 路径迁移

macOS 与 Windows 路径不同，导入时必须处理：

- 原始路径不存在时，标记为 `missing`。
- 如果备份包含真实案件文件夹，恢复到用户选择的新根目录。
- 恢复后重写案件台账里的 `path`。
- 保留旧路径到 `path_history`。

### 2.8 后台任务与进度

备份和导入都必须后台执行。

进度信息：

- 正在收集文件清单。
- 正在写入备份包。
- 正在计算校验。
- 正在导入配置。
- 正在恢复案件文件。
- 完成/失败。

取消策略：

- 备份取消：删除未完成备份包。
- 导入取消：第一版可限制在导入前确认，导入过程中不建议强制取消；或仅在复制案件文件阶段允许取消。

### 2.9 错误处理

需要明确提示：

- 文件被占用。
- 权限不足。
- 备份包损坏。
- 版本不兼容。
- 恢复目标目录空间不足。
- 同名目录冲突。

导入失败时：

- 不应让软件数据处于半恢复状态。
- 覆盖导入前必须先自动备份当前数据。

### 2.10 测试计划

新增测试：

```text
tests/test_backup_manifest.py
tests/test_backup_service.py
tests/test_restore_service.py
```

测试覆盖：

- 创建 manifest。
- 备份配置、模板、案件台账。
- 校验 checksums。
- 读取备份清单。
- 合并导入。
- 覆盖导入前自动备份。
- 路径迁移。
- 文件缺失/权限错误时记录失败清单。

### 2.11 验收标准

备份验收：

- 可从设置页创建备份包。
- 备份包包含 manifest 和 checksums。
- 默认备份配置、模板、案件台账和 sidecar 元数据。
- 可选包含真实案件文件夹。
- 大备份过程 UI 不冻结。

导入验收：

- 可读取备份包信息。
- 可合并导入配置、模板和案件台账。
- 可选择恢复案件文件夹到新目录。
- 导入后案件列表、模板和设置可正常读取。
- 导入失败时有清晰错误报告。
