2.1 案件管理 最迫切

当前工具每次打开都是空白状态。一个律师同时处理 20-50 个案件，信息需要在多个工具间反复搬运。律师实际工作流：

收案 → 委托手续 → 立案 → 举证 → 开庭 → 判决 → 上诉/执行 → 结案归档

建议：案件工作台 — 左侧案件列表（按状态分组），右侧案件详情面板，与现有的模板生成和归档功能打通。

┌─────────────────────────────────────────────────────────────┐ │ 案件工作台 [设置] [归档] │ ├──────────┬──────────────────────────────────────────────────┤ │ 案件列表 │ 案件详情 │ │ │ ┌──────────────────────────────────────────────┐ │ │ ▼ 进行中 │ │ 基本信息（当事人、法院、案号等） │ │ │ 张三v李四│ ├──────────────────────────────────────────────┤ │ │ 王五v赵六│ │ 文书制作（模板列表 + 变量表单） │ │ │ │ ├──────────────────────────────────────────────┤ │ │ ▼ 待立案 │ │ 期限日历 │ │ │ 刘七v孙八│ ├──────────────────────────────────────────────┤ │ │ │ │ 费用记录 │ │ │ ✓ 已结案 │ ├──────────────────────────────────────────────┤ │ │ ... │ │ 案卷文件夹 [打开] [归档] │ │ │ │ └──────────────────────────────────────────────┘ │ └──────────┴──────────────────────────────────────────────────┘

当事人结构化管理：一个案件通常涉及多个当事人、代理人、证人。当前变量系统只支持"单一原告"场景，需要升级为支持"多人多角色"的当事人列表。

法院信息管理：管辖法院、案号、承办法官、书记员、开庭时间地点 — 这些在每次开庭前都要反复查阅的信息需要结构化存储。

📅
2.2 期限管理 律师生命线

律师最怕错过期限。错过上诉期限意味着丧失上诉权，错过举证期限可能导致败诉。目前完全依赖个人记忆和外部日历。

期限类型起算点常见时长法律依据
立案期限法院收到材料7 日民诉法第126条
答辩期限被告收到起诉状15 日 / 30 日民诉法第128条
举证期限收到举证通知不低于 15 日证据规定
一审判决上诉收到判决书15 日民诉法第171条
裁定上诉收到裁定书10 日民诉法第171条
执行申请判决生效2 年民诉法第246条
财产保全知晓财产线索30 日民诉法第104条
管辖权异议提交答辩状期间15 日民诉法第130条

建议方案：

  • 期限计算引擎 — 输入关键日期，自动计算所有关联期限，支持自定义规则
  • 日历视图 — 月/周视图，不同颜色标记紧急程度（绿色正常 / 黄色临近 / 红色已过期）
  • 到期提醒 — 提前 N 天桌面通知，启动时显示"今日到期"列表

为什么不做 Web 端？
痛点场景
文件脱节案件信息在网页上，文件在 D:\案卷\ 里，两边来回切换
上传等待每次查看文件都得上传/下载，大 PDF 传半天
网络依赖法院里没 WiFi、高铁上信号差、当事人办公室网络受限
加载慢打开网页→登录→找案件→找文件→下载→打开，6步才能看文件
隐私风险案件材料上传到第三方服务器，律师保密义务怎么保障？
格式受限Web 预览 Word/PDF 效果差，排版错乱，无法编辑
Web 端流程（6步）：打开浏览器 → 登录 → 找案件 → 找文件 → 下载 → 打开

本地化流程（1步）：点击 → 打开
📋
案件管理："超级书签"而非"管理系统"

不做传统 CRUD 系统。律师已经在用资源管理器管理案卷文件夹了，工具只需要记住路径、快速访问、补充元数据。

案件注册 — 三种方式

自动 生成时自动注册

用工具生成案卷后，文件夹自动出现在案件列表。零额外操作，从文件夹名和变量中提取信息。

手动 添加已有文件夹

对于工具生成前就存在的案卷文件夹，选择文件夹即可注册，自动从文件名解析信息。

批量 扫描目录批量注册

律师按年份组织案卷文件夹，一次扫描即可发现所有案件并批量注册。

期限管理 — 轻量级，不过度设计

核心需求：设一个日期 → 快到了提醒 → 过期了标红。不需要复杂日历、不需要团队协作。

三种期限来源

手动 直接添加

输入标题、截止日期、类型、备注。简单直接。

自动 从关键日期推算

填写立案日期后，自动计算答辩期限(15日)、举证期限(15日)、上诉期限(15日)等。

OCR 从传票提取

OCR 识别法院传票后，自动提取开庭日期并创建期限。

三级提醒机制

级别一：应用内提醒（最重要）

启动时弹窗检查所有期限，案件详情面板颜色标记（红≤7天 / 黄≤30天 / 绿>30天）

级别二：系统通知

Windows Toast 通知，运行中每30分钟检查一次。应用未开不提醒（不做常驻服务）

级别三：日历视图（可选）

简单月视图，彩色圆点标记期限。优先级低于列表视图。
