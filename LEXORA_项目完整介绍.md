## 一、项目背景与定位

### 1.1 项目起源

案件文件夹管理系统（LEXORA）诞生于律师日常办案的真实痛点。在法律实务中，律师每次接手新案件都需要手动创建标准化的案卷文件夹结构，重复填写大量当事人信息，人工整理归档材料。这些机械劳动占据了律师大量时间，而时间本应投入到案件本身的分析与策略制定上。

**传统律师办案流程中的痛点**：

| 痛点 | 描述 |
|------|------|
| 文件夹创建繁琐 | 每个案件需要手动创建 8-10 个子文件夹，命名不统一 |
| 信息重复填写 | 同一当事人的姓名、身份证号等信息在不同文书中反复录入 |
| 模板管理混乱 | Word 文档模板散落在电脑各处，版本不一 |
| 证件信息录入慢 | 身份证、户口簿等证件信息需逐字手动输入 |
| 归档效率低下 | 结案后手动整理材料、编目、标注页码 |
| 期限管理缺失 | 律师最怕错过期限，但完全依赖个人记忆和外部日历 |

### 1.2 核心定位

**"以本地文件夹为核心载体的案件管理桌面应用"**

LEXORA 的核心理念是：**让文件夹本身成为案件管理系统**。不依赖云端、不依赖数据库服务器，律师的电脑里每一个案件文件夹就是一个完整的案件单元，包含所有文书、证据和归档材料。

从"案卷生成工具"起步，逐步升级为"律师案件助手"，最终愿景是：

> **"让律师从机械劳动中解放出来，专注于案件本身。"**

### 1.3 名称由来

- **LEXORA**：Lex（拉丁语"法律"）+ Ora（拉丁语"说话/祈祷"），寓意"法律之声"
- 内部标识 `LawyerCaseTool` 为兼容保留的历史标识，运行时配置目录继续沿用此名称

### 2.2 功能详细说明

#### 2.2.1 案件管理（核心）

**案件工作台**：以案件为中心的统一管理界面，支持：

- **案件台账**：维护本地案件列表，包含标签、状态、路径历史
- **案件状态机**：收案 → 立案 → 审理 → 结案 → 归档
- **案件筛选**：按分类（民事/刑事/行政/非诉/仲裁）、状态、目录状态、标签筛选
- **案件卡片**：直观展示案件基本信息，支持快速跳转
- **案件详情面板**：展示案件基本信息、文书制作、期限提醒、费用记录、案卷文件夹
- **期限管理**：内置法定期限规则库，自动计算关联期限
- **速记功能**：案件备注和待办事项

**对方当事人智能映射**：在被告/被申请人模板中勾选"是否为对方当事人"后，对方姓名直接填入原告/申请人变量。

#### 2.2.2 模板生成与变量替换

**模板系统**：

- **17 个内置模板**：覆盖民事（原告/被告简易+完整）、刑事、行政（原告/被告）、劳动仲裁（申请人/被申请人）、商事仲裁（申请人/被申请人）、非诉案件等
- **模板置顶**：全局最多置顶 3 个，各分类最多置顶 1 个
- **模板管理器**：支持模板的增删改查、分类筛选、搜索
- **自定义模板**：用户可创建、编辑自定义模板

**变量系统**：

- **变量格式**：`{{variable}}`，支持中文变量名
- **变量类型**：文本、数字、日期、下拉选择
- **变量分组**：按业务逻辑分组（当事人信息、案件信息、法院信息等）
- **动态创建**：OCR 识别后自动创建新变量
- **默认变量**：每个模板内置办案阶段、联系方式等通用变量

**文件夹结构预览**：

```
{{case_number}}_{{client_name}}/
├── 0委托手续及程序性材料/
│   ├── 委托合同.docx
│   ├── 授权委托书.docx
│   └── 谈话笔录.docx
├── 1文书材料/
├── 2证据材料/
├── 3法律检索/
├── 4类案检索/
├── 5庭审材料/
├── 6裁判文书/
├── 7执行材料/
└── 8结案材料/
```

#### 2.2.3 OCR 智能识别

**技术架构**：基于 RapidOCR（ONNX Runtime）实现轻量级 OCR 识别

- 体积小（40MB vs PaddleOCR 的 200MB+）
- 启动快，识别稳定
- 模型内置，无需额外下载

**识别流程**：

```
添加证件图片/PDF → OCR 文字识别 → 文档解析器解析字段 → 字段匹配到模板变量 → 自动填充表单
```

**支持的证件类型**：

| 证件类型 | 状态 | 提取字段 |
|---------|------|---------|
| 身份证（正面） | ✅ 已实现 | 姓名、性别、民族、出生日期、住址、身份证号 |
| 身份证（反面） | ✅ 已实现 | 签发机关、有效期限 |
| 户口簿 | ⚠️ 框架搭建 | 基本结构已定义 |
| 护照 | ⚠️ 框架搭建 | 基本结构已定义 |
| 驾驶证 | ⚠️ 框架搭建 | 基本结构已定义 |
| 营业执照 | ⚠️ 框架搭建 | 基本结构已定义 |

**识别特性**：

- 批量添加图片/PDF 文件
- 进度条显示识别进度
- 置信度指示器（高/中/低颜色标识）
- 低置信度字段红色警告
- 识别结果可手动修正
- 支持导出为 JSON/Excel/Word
- 多行地址识别（如住址跨行）

#### 2.2.4 电子化归档

**三栏布局设计**：

- **左侧**：变量定义面板（280px），支持导入/导出 JSON，与主界面模板变量系统同步
- **中间**：文件预览区（自适应），支持 Word/PDF/图片预览，变量高亮显示
- **右侧**：文件夹结构树（300px），📁📄🖼️ 图标系统

**核心功能**：

- 选中文字右键"设置为变量"
- 变量高亮（绿色背景）
- 保存/另存为（格式保留）
- 图片预览缩放控件
- 变量面板拖拽排序与卡片化

#### 2.2.5 截图合并 PDF

**应用场景**：将微信聊天记录截图等图片批量合并为 PDF

**功能特性**：

- 图片导入：支持拖放文件夹/文件、点击选择
- 排序规则：手动拖拽排序、正序/倒序（按文件名数字排序）
- 布局设置：每页 1/2/3 张图片，横向/纵向 A4
- 标签系统：自动编号/自定义前缀/文件名，5 档文字大小
- 保存方式：保存到源文件夹或弹出保存对话框
- 视图切换：缩略图视图与列表视图

#### 2.2.6 Word 模板制作器

**核心功能**：让用户自行制作 Word 模板，上传到分类文件夹并进行变量替换

- 三栏布局：左侧文件浏览器、中间 Word 预览区、右侧变量面板
- 工具栏：打开、替换选中、全部替换、撤销、保存、另存为
- Word 预览：python-docx 文本预览 + 变量高亮
- 格式保持的文本替换（Run 级别操作）
- 右键撤销变量功能
- 智能变量推荐
- 自定义变量添加（与模板管理器变量定义深度整合）
- 文件操作：单击重命名、双击打开、展开/收缩

#### 2.2.7 Word 自动排版

参照 GB/T 9704-2012 标准，智能识别标题层级并应用法律文书格式。

#### 2.2.8 法院短信增强

支持拖拽 PDF/图片自动识别传票，提取被传唤人、开庭信息并加入期限。

#### 2.2.9 工作台仪表盘（v2.0）

**从"只能看"变成"真正能点、能跳、能操作"**：

- 快捷操作全面重映射（新建案件、新建期限、法院短信、OCR识别等）
- Dashboard 图表交互（横向柱状图 hover 高亮、分类点击跳转）
- 案件中心 UI 优化
- 顶部工具栏修复

#### 2.2.10 本地文件搜索

基于内存索引的全文搜索服务：

- `FileIndexStore`：索引存储
- `FileIndexer`：文件索引器
- `FileSearchService`：搜索服务
- 支持文件名和内容搜索

### 3.5 关键设计模式

| 设计模式 | 应用场景 |
|---------|---------|
| **单例模式** | `ConfigManager`、`PathManager`、`LoggerManager`、`Application` — 线程安全的双重检查锁定 |
| **策略模式** | 验证器根据变量类型选择不同验证策略；OCR 解析器根据证件类型选择不同解析策略 |
| **观察者模式** | 进度回调、信号/槽机制 |
| **模板方法模式** | 验证器基类定义接口，子类实现具体逻辑 |
| **工厂模式** | 各模块通过 `get_xxx()` 工厂函数获取实例 |

### 3.6 数据流程

#### 案卷生成流程

```
选择模板 → 提取变量定义 → 动态生成输入控件 → 用户填写变量
    → 预览文件夹结构 → 生成文件夹 + 处理 Word 模板 → 完成/打开
```

#### OCR 识别流程

```
添加证件图片/PDF → RapidOCR 文字识别 → 文档解析器解析字段
    → 字段匹配到模板变量 → 自动填充表单 → 用户确认
```

#### 电子化归档流程

```
选择案卷文件夹 → 浏览文件结构 → 提取变量信息
    → 选择归档模板 → 替换导出
```

## 五、使用方法

### 5.1 环境准备

**系统要求**：
- Windows 10+ 或 macOS
- Python 3.10+

### 5.2 安装步骤

```bash
# 克隆仓库
git clone <repository-url>
cd LawyerCaseTool

# 创建虚拟环境（推荐）
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS
source .venv/bin/activate

# 安装核心桌面依赖（双平台）
pip install -r requirements.txt

# 如需 OCR 功能，额外安装
pip install -r requirements-ocr.txt
```

### 5.3 macOS + OCR 环境

```bash
bash scripts/setup_macos_ocr_env.sh
source .venv-ocr311/bin/activate
python src/main.py
```

或直接运行：

```bash
bash scripts/run_macos_ocr.sh
```

### 5.4 启动应用

```bash
python src/main.py
```

### 5.5 命令行参数

| 参数 | 说明 |
|------|------|
| `--directory, -d` | 指定输出目录 |
| `--template, -t` | 指定模板 ID |
| `--benchmark-startup` | 输出启动性能 JSON |
| `--import-backup` | 导入 LEXORA 备份文件 |

### 5.6 典型使用流程

#### 流程一：创建新案件文件夹

1. 启动应用，在工作台选择"创建中心"
2. 从模板列表选择合适的模板
3. 填写变量（案号、委托人姓名等）
4. 预览文件夹结构
5. 点击"生成案卷"

#### 流程二：OCR 识别证件信息

1. 选择模板后，点击"信息识别"按钮
2. 添加身份证、户口簿等证件图片或 PDF
3. 点击"开始识别"
4. 核对识别结果（低置信度字段标红提醒）
5. 点击"应用到案卷变量"自动填充表单

#### 流程三：电子化归档

1. 在工具中心选择"电子化归档"
2. 选择案卷文件夹
3. 浏览文件、提取变量、设置变量
4. 选择归档模板，替换导出

#### 流程四：截图合并 PDF

1. 在工具中心选择"截图合并"
2. 导入图片（拖放或选择文件夹）
3. 调整排序和布局
4. 生成 PDF

python scripts/install_context_menu.py
